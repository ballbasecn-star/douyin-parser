# 产品需求总览

## 产品摘要

`douyin-parser` 最初是一个单视频解析工具，现在正在演进为一个面向抖音创作者的“博主监控与内容情报系统”。

当前稳定能力包括：

- 解析单条抖音视频
- 获取视频元数据与统计数据
- 本地或云端转录视频语音
- 使用 AI 分析视频文案结构
- 通过 Web UI、流式 API 和同步 API 对外提供能力

下一阶段的产品方向是：支持维护一组固定的对标博主主页，拉取其视频列表，并允许用户手动选择某些视频进行深度分析。

这部分的详细功能文档位于：

- `docs/product/creator-monitoring-v1.md`

## 目标用户

- 研究同领域内容的抖音创作者
- 希望积累同行素材库的内容运营人员
- 希望把同行视频转成结构化数据的小团队

## 用户核心任务

1. 解析一条视频并理解它为什么有效
2. 维护一组对标博主
3. 将对标博主的视频列表沉淀成结构化素材库
4. 挑选值得深入分析的视频
5. 将分析结果用于后续生成与策划

## 当前范围

### 当前已在范围内

- 单视频解析
- 单视频转录
- 单视频 AI 分析
- 供 agent 调用的同步 API
- 基于 Docker 的部署

### 下一阶段计划范围

- 基于主页分享链接添加博主
- 提取并保存稳定的博主标识
- 同步博主视频列表并存储
- 手动选择视频执行转录与分析

## 第一版监控功能的非目标

- 自动分析所有博主的所有视频
- 复杂任务队列基础设施
- 多用户权限系统
- 自动周报或情报报告
- 大规模历史回溯

## 成功标准

如果第一版监控功能满足以下条件，就算成功：

1. 用户可以从主页分享链接添加博主
2. 系统能保存稳定的博主标识
3. 系统可以同步并展示博主最近的视频列表
4. 用户可以选择一条视频并执行已有的解析/转录/分析流程
5. 结果可以持久化保存并再次查看

## 产品约束

- 抖音 Web API 与签名规则可能变化
- Cookie 的可用性是稳定抓取的硬依赖
- 转录与 AI 分析对单条视频都有时间和成本开销
- 新监控功能不能破坏现有单视频工作流
