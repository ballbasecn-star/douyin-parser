# ADR 0001：博主监控第一版先采用“手动深度分析”

## 状态

已接受

## 背景

产品正在从“单视频解析工具”扩展为“博主监控系统”。这时候有两条明显路线：

1. 同步博主视频后，自动分析全部视频
2. 先同步博主视频列表，再让用户手动选择需要分析的视频

第一条路线听起来更自动化，但会显著增加成本、执行时长、队列复杂度和运维风险，而这时博主同步层本身还没有稳定。

## 决策

博主监控第一版只做以下事情：

- 管理对标博主
- 同步博主视频列表
- 持久化保存视频基础指标
- 允许用户手动选择某些视频执行转录与 AI 分析

第一版不自动分析所有同步到的视频。

## 结果影响

### 正面影响

- 实现复杂度更低
- 转录和 AI 成本更可控
- 更快交付出可用版本
- 能保留现有单视频内核作为稳定基础能力

### 负面影响

- 第一版自动化程度较低
- 用户仍需手动挑选值得分析的视频

## 后续

如果后续博主视频同步稳定、并且自动分析需求明确，再通过新的 ADR 引入：

- 定时同步
- 选择性自动分析规则
- 队列与失败重试基础设施
