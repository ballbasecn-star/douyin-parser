# 当前路线图

## 当前状态

当前仓库已经支持：

- 单视频解析
- 转录
- AI 分析
- 面向 agent 的同步 API
- 基于共享 nginx 的 Docker 部署

## 当前优先级

当前已完成“第二阶段目录迁移”和旧重复代码清理，并已开始“博主监控 V1”第一批后端开发。

## 当前工程工作

### 第二阶段目录迁移

当前迁移目标：

1. 让 `app/` 成为后端主目录
2. 将现有 `douyin/` 中的稳定内核按风险可控的顺序迁入 `app/`
3. 每完成一层迁移，都用现有测试回归验证

当前已完成：

1. `VideoInfo` 已迁移到 `app/domain/`
2. Cookie 存储与 Webhook 已迁移到 `app/infra/`
3. 抖音请求客户端与签名基础设施已迁移到 `app/infra/`，抓取服务已迁移到 `app/services/`
4. 转录、分析和主解析流程已迁移到 `app/services/`
5. CLI 入口已迁移到 `app/cli/`

当前已完成补充说明：

1. 旧目录清理与兼容层收口已完成
2. 仓库内部实现已统一收口到 `app/`

## 博主监控 V1

### 目标

1. 通过主页分享链接添加博主
2. 保存稳定的博主标识
3. 同步博主视频列表并保存基础指标
4. 允许手动选择视频执行转录和分析

### 暂不纳入范围

- 自动批量分析
- 后台定时同步
- 历史趋势看板
- 自动报告生成

### 当前已完成

1. 已引入持久化层基础设施，支持 SQLite 本地开发与 PostgreSQL 生产接入
2. 已建立 `creators`、`creator_videos`、`video_analyses` 三类数据模型
3. 已新增博主管理、视频同步、手动分析的后端服务与 API
4. 已补充回归测试，现有功能与新能力可共同运行

### 当前下一步

1. 将博主管理与视频列表接入现有前端页面
2. 补充主页链接解析与博主列表同步的真实链路验证
3. 为持久化层补充迁移方案与生产环境配置说明

## 当前推荐的实现顺序

1. 将主页链接解析与稳定博主标识提取做真实链路验证
2. 完成博主管理与视频列表的前端接入
3. 补充生产环境 PostgreSQL 配置与迁移方案
4. 将被选中的视频重新接回现有单视频内核的页面工作流

## 当前需要持续关注的风险

- 博主主页接口的稳定性
- Cookie 的时效性
- 在开始积累数据前先定好数据结构
- 运行时配置与会话数据的安全性
