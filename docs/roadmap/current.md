# 当前路线图

## 当前状态

当前仓库已经支持：

- 单视频解析
- 转录
- AI 分析
- 面向 agent 的同步 API
- 基于共享 nginx 的 Docker 部署

## 当前优先级

当前先完成“第二阶段目录迁移”，再进入“博主监控 V1”。

## 当前工程工作

### 第二阶段目录迁移

当前迁移目标：

1. 让 `app/` 成为后端主目录
2. 将现有 `douyin/` 中的稳定内核按风险可控的顺序迁入 `app/`
3. 每完成一层迁移，都用现有测试回归验证

当前已完成：

1. `VideoInfo` 已迁移到 `app/domain/`
2. Cookie 存储与 Webhook 已迁移到 `app/infra/`
3. 抖音请求客户端与签名基础设施已迁移到 `app/infra/`，抓取服务已迁移到 `app/services/`
4. 转录、分析和主解析流程已迁移到 `app/services/`

当前仍待完成：

1. CLI 入口迁移与旧目录清理

## 博主监控 V1

### 目标

1. 通过主页分享链接添加博主
2. 保存稳定的博主标识
3. 同步博主视频列表并保存基础指标
4. 允许手动选择视频执行转录和分析

### 暂不纳入范围

- 自动批量分析
- 后台定时同步
- 历史趋势看板
- 自动报告生成

## 当前推荐的实现顺序

1. 引入持久化层
2. 增加博主数据模型与存储
3. 实现主页链接解析与稳定博主标识提取
4. 实现博主视频列表同步
5. 增加博主管理的 Web/API 接口
6. 将被选中的视频重新接回现有单视频内核

## 当前需要持续关注的风险

- 博主主页接口的稳定性
- Cookie 的时效性
- 在开始积累数据前先定好数据结构
- 运行时配置与会话数据的安全性
